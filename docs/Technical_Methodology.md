# Technical Methodology & Architecture
**Project**: Forensic-AI V2.1  
**Objective**: Cross-Dataset Generalization and Court-Admissible Forensic Calibration (CLLR < 0.4)

---

## 1. Architectural Design

Our model, `ForensicModel`, is a multi-branch neural network designed to capture spatial artifacts, temporal inconsistencies, and compression-invariant frequency anomalies.

### 1.1 Modality Branches
1. **Spatial Branch (CLIP-ViT Backbone)**
   - **Base**: `vit_base_patch16_clip_224.openai` (Timm).
   - **Resolution**: Interpolated to 256x256.
   - **Constraint**: Patch embeddings and transformer blocks 0–7 are strictly frozen. Only the top 4 blocks are trainable. This prevents catastrophic forgetting and catastrophic overfitting to source-domain (FF++) spatial artifacts.
   - **Projection**: $768 \rightarrow 512$ dimensions.
2. **Frequency Branch (DCT)**
   - **Mechanism**: Extracts 2D Discrete Cosine Transform (DCT) coefficients from the isolated Y, Cb, and Cr channels to detect transposed high-frequency spectral artifacts generated by GANs/Diffusion models.
   - **Projection**: $256 \rightarrow 512$ dimensions.
3. **Temporal Module (Self-Attention)**
   - **Input Fusion**: Spatial and Frequency projections are concatenated ($512 + 512 = 1024$ dimensions).
   - **Sequence Processing**: A 4-head Self-Attention layer aggregates $T=16$ temporal frames into a permutation-invariant feature vector, capturing inter-frame jitter mathematically without recurrent overhead.
4. **Classification Head**
   - A multi-layer perceptron (MLP) consisting of `Linear(1024, 512)`, `BatchNorm1d`, `ReLU`, `Dropout(0.3)`, and final projection `Linear(512, 1)`.

## 2. Training Protocol (Domain Adaptation Pipeline)

We enforced a strict, multi-stage curriculum to enforce out-of-distribution (OOD) generalization without data leakage.

### 2.1 Stage 1: Base Optimization
- **Dataset**: FaceForensics++ (FF++) exclusively.
- **Objective**: BCEWithLogitsLoss.
- **Convergence**: Achieved Validation AUC > 0.95. The model learned fundamental manipulation artifacts but remained highly biased to the FF++ domain distributions.

### 2.2 Stage 2: Hybrid Domain Adaptation
- **Datasets**: 50/50 balanced sampling of FF++ and WildDeepfake.
- **Goal**: Regularize the decision boundary using highly diverse, uncontrolled internet data (WildDeepfake) to reduce the model's reliance on dataset-specific compression signatures.
- **Augmentation**: Real-time CRF-35 JPEG compression simulation was utilized to simulate destructive downsampling.
- **Early Stopping Target**: Celeb-DF Proxy Set (strictly disjoint from test and calibration sets). Best epoch chosen based on Celeb-DF zero-shot AUC.

## 3. The Calibration Bottleneck & Solution

### 3.1 The Problem
Evaluating Stage 2 weights via the standard Likelihood Ratio (LR) protocol resulted in severe forensic unreliability on unseen domains:
- **FF++ CLLR**: 0.23 (Excellent, highly calibrated locally).
- **Celeb-DF CLLR**: 0.70 (Mathematically uncalibrated).
- **WildDeepfake CLLR**: 1.48 (Severely penalizing cost).

The model exhibited **overconfidence bias**. Because single-domain affine calibration (Platt Scaling / Isotonic Regression) relies on the logit variance of the *source* dataset, it mapped out-of-distribution logits to extreme probabilities ($P \approx 0.99$ or $0.01$), heavily inflating the Cost of Log-Likelihood Ratio (CLLR) penalty.

### 3.2 The Fix: Multi-Domain Hybrid Calibration
We rewrote the calibration pipeline (`src/calibration/run_calibration.py`) to support disjoint multi-domain anchoring:
1. **Partitioning**: Carved out strict, leakage-free `cal` splits from FF++ ($n=132$), Celeb-DF ($n=494$, balanced 247/247 via determinism), and WildDeepfake ($n=50$, balanced 25/25).
2. **Temperature Scaling**: Fit a global scalar ($T=1.3315$) via L-BFGS to map logits to mathematically neutral probabilities.
3. **Isotonic Regression (PAVA)**: Fit a non-parametric monotonic step function over the concatenated, multi-domain logit space, successfully preventing the model from generating extreme log-odds for OOD samples.

## 5. Conclusion
- **Hypothesis Validated**: Spatial-only (ViT) representation suffers from critical OOD generalization degradation. Combining frequency spectral anomaly detection (DCT) with temporal sequencing (SA) dramatically improves cross-domain robustness.
- **Calibration Status**: Integrating a strictly disjoint, multi-domain calibrator rectifies overconfidence bias, reducing Celeb-DF CLLR from 0.70 to a court-admissible **0.37**. 
- **Methodological Integrity**: Zero data leakage was maintained across the $Train \rightarrow Proxy \rightarrow Calibrate \rightarrow Test$ pipeline.
